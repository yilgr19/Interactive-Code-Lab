¿Qué es un algoritmo?

Un algoritmo es una secuencia finita de pasos bien definidos que resuelve un problema. En programación, traducimos ese flujo en instrucciones (entrada, decisiones, repetición, salida). Las estructuras condicionales son el punto donde el flujo se bifurca según una condición (como en el reto de validación por edad).

Abajo tienes el archivo FlujoLogicoAlgoritmo.glb: no es una figura “vacía”, sino un diagrama de flujo en 3D donde cada forma y color tiene un significado fijo (como en papel, pero rotable y en RA). Regenerable con scripts/generar_modelo_flujo_logico.py.

Sobre la RA: carga primero el 3D; en móvil, “Escanear en mi espacio” abre la escena en tu entorno (Chrome en Android, Safari en iOS). Los puntos interactivos sobre el modelo repiten la explicación al pulsarlos.

Cómo leer el FlujoLogicoAlgoritmo

Sigue el flujo de arriba hacia abajo, como al leer un algoritmo línea a línea. Luego mira las dos salidas del rombo: una es el camino cuando la condición se cumple, la otra cuando no.

  1. Disco cyan (arriba) — Inicio del proceso. Aquí “entra” el dato que vas a evaluar (en el reto: la edad de la persona).
  2. Formas grises — Conectores y flecha: el control baja hacia la decisión (orden de ejecución).
  3. Rombo ámbar (centro) — Condición / pregunta lógica. En código es el if (…). En el reto, la pregunta es: ¿edad ≥ 18?
  4. Rectángulo verde (izquierda) — Rama verdadera (la condición se cumple): acceso permitido → equivale a la rama del if.
  5. Rectángulo rojo (derecha) — Rama falsa (no se cumple): acceso denegado → equivale a la rama del else.
  6. Base oscura (abajo) — Suelo del diagrama; indica que ambos caminos ya terminaron esa validación (en un programa podría seguir otro bloque después).

Equivalencia rápida

if (edad >= 18) → bloque verde
else → bloque rojo

Leyenda de colores (igual que en el 3D)

  • Inicio / entrada de datos
  • Decisión (rombo)
  • Condición verdadera
  • Condición falsa

Unidad 2 · Estructuras condicionales

Sistema de validación de acceso por edad

Implementa la lógica: si la persona tiene 18 años o más, el acceso es permitido; si no, denegado. Es el mismo patrón que el rombo de decisión del modelo 3D en Algoritmos + RA. Puedes escribir en JavaScript o pseudocódigo (español) con si / si no.

Ejemplo válido (una de varias formas):

if (edad >= 18) {
  console.log("Permitido");
} else {
  console.log("Denegado");
}
