    Nivel 1 · Fundamentos — resumen detallado con ejemplos (1.1 a 1.3)

    Qué cubre esta unidad. Tres bloques que siempre van juntos: algoritmo y bifurcación (if/else), datos con nombre y tipo (variables), y el esquema entrada → proceso → salida para organizar cualquier ejercicio pequeño. Es el cimiento del Nivel 2 y del resto del curso.

    1.1 · Algoritmos y condicionales (if / else, diagramas)

    Visto en clase: un algoritmo es una lista finita de pasos sin ambigüedad; en un diagrama de flujo el rombo representa la pregunta sí/no. En código, eso es un condicional: si la condición es verdadera se ejecuta un bloque; si no, otro (else). La ejecución sigue siendo línea a línea, pero la rama depende de los datos.

    JavaScript — dos ramas según la edad (idea general)

    let edad = 17;

if (edad >= 18) {
  console.log("Mayor de edad");
} else {
  console.log("Menor de edad");
}

    JavaScript — mismo patrón que el reto «acceso por edad» (18 años)

    let edad = 20;

if (edad >= 18) {
  console.log("Acceso permitido");
} else {
  console.log("Acceso denegado");
}

// Prueba mental con edad = 17: debe caer en la segunda rama

    Pseudocódigo — misma lógica en español

    entero edad
edad <- 17

si edad >= 18 entonces
  escribir "Mayor de edad"
si no
  escribir "Menor de edad"
fin si

    Reto alineado: condicional por edad con mensajes claros de permitido/denegado y umbral 18 (como en el laboratorio).

    Errores típicos: olvidar una de las dos ramas; usar = (asignación) en lugar de >= o === en la condición; no encerrar el bloque entre llaves { } cuando hay varias líneas.

    1.2 · Variables y tipos (let, const, identificadores)

    Visto en clase: una variable es un nombre para un valor en memoria; el identificador debe ser válido (p. ej. camelCase en JavaScript). Los tipos básicos que manejas aquí son número, cadena (texto entre comillas) y booleano. Con let puedes cambiar el valor; con const no reasignas el nombre a otra cosa.

    JavaScript — tres tipos en un mismo programa

    let puntos = 100;                 // número
const nombreCurso = "Lógica";     // cadena (const: no reasignas el nombre)
let aprobado = false;             // booleano

puntos = puntos + 10;             // el valor puede cambiar
console.log(nombreCurso, puntos, aprobado);

    JavaScript — precio × cantidad (mismo enfoque que el reto del laboratorio)

    let precio = 10;
let cantidad = 2;
let total = precio * cantidad;

console.log(total);   // 20

    Pseudocódigo — tipos explícitos

    entero puntos
cadena nombreCurso
logico aprobado

puntos <- 100
nombreCurso <- "Lógica"
aprobado <- falso

puntos <- puntos + 10
escribir nombreCurso, puntos, aprobado

    Reto alineado: declarar precio y cantidad, calcular el total con * y dejar visible el 20 (consola o comentario).

    Errores típicos: poner números entre comillas y tratarlos como texto sin querer; olvidar declarar antes de usar; nombres ilegibles o con espacios.

    1.3 · Entrada, proceso y salida (console.log, flujo E → P → S)

    Visto en clase: separar el enunciado en tres fases: qué datos entran, qué operaciones o decisiones forman el proceso, y qué sale (en el navegador, casi siempre con console.log al practicar). Las variables de 1.2 guardan entradas y resultados intermedios; un if de 1.1 puede formar parte del proceso.

    JavaScript — rectángulo: entrada → proceso → salida

    // Entrada (podrían venir de un formulario más adelante)
let base = 4;
let altura = 5;

// Proceso
let area = base * altura;

// Salida
console.log("Área del rectángulo:", area);  // 20

    JavaScript — E/P/S con decisión en el proceso (integra 1.1 + 1.3)

    // Entrada
let edad = 16;

// Proceso (aquí va la lógica que transforma o clasifica)
let puedeVotar;
if (edad >= 18) {
  puedeVotar = true;
} else {
  puedeVotar = false;
}

// Salida
console.log("¿Puede votar?", puedeVotar);  // false

    Pseudocódigo — área con el mismo flujo

    entero base, altura, area
base <- 4
altura <- 5
area <- base * altura
escribir "Área:", area

    Reto alineado: área del rectángulo con base 4, altura 5, producto y 20 visibles, más console.log.

    Errores típicos: mezclar salida sin haber calculado antes en el proceso; olvidar comentar o nombrar qué parte es E, P o S en ejercicios largos; perder el * en una fórmula geométrica.

    Ideas que deben ir juntas: el if siempre evalúa una expresión verdadera/falsa; las variables son los “cubos” donde guardas lo que entra y lo que vas calculando; pintar mentalmente entrada → proceso → salida evita saltarte pasos cuando el enunciado mezcla texto, números y decisiones.

    Nivel 2 · Expresiones, decisiones, texto y bucles — resumen detallado con ejemplos (2.1 a 2.4)

    Qué cubre esta unidad. Cuatro bloques que se usan juntos en programas reales: calcular y comparar (operadores), elegir entre varias opciones (else if), trabajar texto (longitud, concatenar, índices) y repetir (while / for) sin duplicar líneas.

    2.1 · Operadores y expresiones

    Visto en clase: aritmética (+ - * /), módulo %, comparaciones (===, <, etc.) y lógica (&&, ||). Las expresiones se evalúan a un valor; los paréntesis fijan el orden.

    JavaScript — aritmética, descuento y comparación

    let precio = 100;
let descuento = 0.15;
let total = precio * (1 - descuento);   // paréntesis primero

let a = 10, b = 3;
console.log(a + b);    // 13
console.log(a % b);    // 1  (resto: útil para par/impar)

console.log(10 === 10);           // true
console.log(10 === "10");         // false (tipos distintos)
console.log(a > b && a % b === 1);  // true

    JavaScript — par o impar con %

    let n = 8;
if (n % 2 === 0) {
  console.log(n + " es par");
}
n = 7;
if (n % 2 !== 0) {
  console.log(n + " es impar");
}

    Reto alineado: usar % y condicionales para 8 (par) y 7 (impar).

    2.2 · Decisiones múltiples (else if, anidación)

    Visto en clase: más de dos salidas encadenando condiciones; el orden de los if / else if importa; el último else atrapa el resto. También viste la idea de if anidado para decisiones en dos etapas.

    JavaScript — nota numérica a letra (tres tramos)

    let nota = 85;

if (nota >= 90) {
  console.log("Letra A");
} else if (nota >= 70) {
  console.log("Letra B");   // 85 cae aquí
} else {
  console.log("Letra C");
}

    Pseudocódigo — misma estructura

    entero nota
nota <- 85

si nota >= 90 entonces
  escribir "Letra A"
si no si nota >= 70 entonces
  escribir "Letra B"
si no
  escribir "Letra C"
fin si

    Reto alineado: nota 85 con umbrales 90 y 70 y mensaje que deje clara la B.

    2.3 · Cadenas de texto (strings)

    Visto en clase: literales entre comillas; propiedad .length; acceso por índice [0]; concatenación con +. El texto es un tipo más junto a números y booleanos.

    JavaScript — longitud, unir textos, primer carácter

    let saludo = "Hola";
let nombre = "Ana";

console.log(saludo.length);              // 4  (H-o-l-a)
console.log(saludo + ", " + nombre);    // "Hola, Ana"
console.log(saludo[0]);                  // "H"

// Cuidado: espacios y signos cuentan en length
console.log("a b".length);               // 3

    Reto alineado: "Hola", .length y dejar visible el 4.

    2.4 · Bucles (while y for)

    Visto en clase: iteración = repetir un bloque; en while la condición se evalúa antes de cada vuelta; en for llevas inicio, condición y paso en la cabecera; acumuladores (suma) empiezan en 0.

    JavaScript — while: cuentas del 1 al 3

    let n = 1;
while (n <= 3) {
  console.log("Vuelta:", n);
  n = n + 1;   // o n++;
}

    JavaScript — for y suma del 1 al 10 (reto del laboratorio: total 55)

    let suma = 0;
for (let i = 1; i <= 10; i++) {
  suma = suma + i;   // o suma += i;
}
console.log(suma);   // 55

    Pseudocódigo — idea equivalente

    entero suma, i
suma <- 0
para i desde 1 hasta 10 hacer
  suma <- suma + i
fin para
escribir suma

    Errores típicos: olvidar i++ en while (bucle infinito); usar < cuando el enunciado incluye el tope (<=); no inicializar suma en 0.

    Ideas que deben ir juntas: una condición en un if o while es siempre una expresión que da verdadero/falso; dentro del bucle usas las mismas variables y operadores que ya practicaste en 2.1.

    Nivel 3 · Estructuras, modularidad y trazado — resumen detallado con ejemplos (3.1 a 3.3)

    Qué cubre esta unidad. Pasas de “unas pocas variables” a colecciones ordenadas (arreglos), aprendes a empaquetar lógica reutilizable en funciones con return, y cierras con el hábito de trazar y comentar: leer código como la máquina, fila a fila, para encontrar errores antes de ejecutar o depurar más rápido.

    3.1 · Arreglos (listas ordenadas)

    Visto en clase: literal [a, b, c]; índices desde 0; length = cantidad de elementos; recorrido con for y i < arr.length. Modificar una casilla no cambia el nombre del arreglo, solo su contenido.

    JavaScript — crear, leer, cambiar y recorrer

    let notas = [8, 9, 10];

console.log(notas[0]);       // 8  (primera posición)
console.log(notas.length);   // 3

notas[1] = 10;               // subo la segunda nota
console.log(notas);          // [8, 10, 10]

for (let i = 0; i < notas.length; i++) {
  console.log("Índice", i, "→", notas[i]);
}

    JavaScript — promedio de tres números (reto del laboratorio)

    let notas = [8, 9, 10];
let suma = notas[0] + notas[1] + notas[2];
let promedio = suma / 3;
console.log(promedio);   // 9

    Pseudocódigo — índice base 0

    arreglo notas de enteros
notas <- [8, 9, 10]
entero i
para i desde 0 hasta longitud(notas) - 1 hacer
  escribir "Casilla", i, ":", notas[i]
fin para

    Errores típicos: usar índice 3 en un arreglo de 3 elementos (válidos 0,1,2); confundir length con el último índice.

    3.2 · Funciones y modularidad

    Visto en clase: definición con nombre y parámetros; cuerpo entre llaves; return devuelve un valor al lugar de la llamada; los argumentos son los valores concretos que pasas al invocar. Las variables locales no ensucian el programa global.

    JavaScript — función que devuelve el doble (reto del laboratorio)

    function doble(x) {
  return x * 2;
}

console.log(doble(3));   // 6
let y = doble(5);
console.log(y);          // 10

    JavaScript — función con varias líneas y uso del resultado

    function sumar(a, b) {
  let s = a + b;
  return s;
}

let total = sumar(10, 3);
console.log("Total:", total);   // 13

    Pseudocódigo — función con retorno

    función sumar(a, b): entero
  retornar a + b
fin función

entero total
total <- sumar(10, 3)
escribir total

    Idea clave: la llamada doble(3) es una expresión que vale 6; puedes anidarla en otras expresiones o pasarla a otras funciones.

    3.3 · Pruebas mentales, trazado y comentarios

    Visto en clase: los comentarios // no se ejecutan; sirven para explicar supuestos, recordar el enunciado o anotar salidas esperadas. La traza es simular la ejecución: tras cada línea, anotas valores de variables y lo que imprimiría console.log.

    JavaScript — código con comentarios y traza mental

    // Traza esperada: imprime 0, luego 1 (dos líneas)
let i = 0;
while (i < 2) {
  console.log(i);
  i++;
}
// Tras el bucle, i vale 2; no se entra otra vez porque 2 < 2 es falso

    Ejemplo de tabla de traza (a mano en papel)

    Línea / vuelta | i (antes) | condición i<2 | salida console | i (después)
Inicio         | 0         | verdadera     | 0             | 1
Vuelta 2       | 1         | verdadera     | 1             | 2
Fin            | 2         | falsa         | (no imprime)  | —

    Reto alineado: while con i desde 0, condición típica i < 2, salida 0 y 1. Es el mismo patrón mental que usarás al depurar bucles en cualquier lenguaje.

    Cierre del curso lógico en este laboratorio: ya puedes combinar datos múltiples (arreglos), abstracción (funciones) y verificación (trazas y comentarios). Lo siguiente en una carrera suele ser más sintaxis, más librerías o otros paradigmas — la lógica que practicaste aquí sigue siendo el núcleo.

    Nivel 1 · 1.1 — Algoritmos, diagramas de flujo y condicionales

    Imagina una receta de cocina: hay ingredientes, pasos en orden y un resultado. En computación, eso es un algoritmo: una lista finita y clara de pasos para que la computadora (o una persona) resuelva un problema sin ambigüedad. No hace falta saber electrónica: solo que la máquina ejecuta instrucciones una tras otra, y a veces debe elegir entre dos caminos según una condición.

    Un diagrama de flujo dibuja esos pasos con formas: óvalo o inicio, flechas, rombo = pregunta sí/no, rectángulos = acciones. El modelo 3D es la misma idea: entrada → decisión → dos salidas.

    ¿Qué es un condicional (if / else)?

    Es una pregunta lógica. Si la respuesta es “sí”, se hace un bloque de instrucciones; si es “no”, se puede hacer otro bloque (else). Así el programa no hace siempre lo mismo: se adapta a los datos.

    Ejemplo en JavaScript (validación por edad)

    let edad = 17;  // dato de entrada (podría venir de un formulario)

if (edad >= 18) {
  console.log("Acceso permitido");
} else {
  console.log("Acceso denegado");
}

    Lectura: “si edad es mayor o igual a 18, muestra permitido; si no, muestra denegado”. El símbolo >= es una comparación, no una asignación.

    Mismo idea en pseudocódigo (español)

    entero edad
edad <- 17

si edad >= 18 entonces
  escribir "Acceso permitido"
si no
  escribir "Acceso denegado"
fin si

    Leyenda del modelo 3D (qué ver en RA)

    1. Discos cyan arriba — bandeja de entrada: aquí “depositas” el dato (ej. edad) antes de decidir.
    2. Cilindro + cono gris — el programa baja en orden hasta llegar a la pregunta.
    3. Rombo grande + punto claro — la condición (if): ¿se cumple o no?
    4. Bloque verde + marcas claras en V — camino SÍ (rama verdadera). Bloque rojo + aspas — camino NO (else).

    Cargando modelo 3D…

    Gira el modelo: la historia va de arriba a abajo. Toca los puntos para repasar cada parte. En RA, coloca la escena en tu mesa y sigue el camino con el dedo.

    Nivel 1 · 1.2 — Variables, identificadores y tipos de datos

    La memoria de la computadora guarda millones de valores. Para no referirnos a “el valor que está en la posición 4 847 291”, usamos un nombre simbólico: eso es una variable. Piensa en una etiqueta pegada a una cajita: el nombre es lo que escribes en el código; dentro de la cajita está el valor actual (que puede cambiar con el tiempo).

    El identificador es el nombre de la variable (por ejemplo edad, nombreUsuario). Convención habitual en JavaScript: camelCase (primera palabra minúscula, siguientes con mayúscula inicial), sin espacios ni acentos, y no puede empezar por número.

    Tipos de dato (qué “forma” tiene el valor)

    • Número — entero (42) o decimal (3.14). En muchos lenguajes se distingue int / float; en JavaScript suele usarse number para ambos.
    • Cadena (string) — texto entre comillas: "Hola" o 'Ana'.
    • Booleano — solo dos valores lógicos: true (verdadero) o false (falso). Sirve para encender/apagar decisiones.

    El modelo 3D usa tres formas distintas para que recuerdes los tipos sin memorizar solo colores: cilindro alto con tapa = número (como una lata medible); lámina ancha y baja con líneas = texto escrito; palanca sobre un poste = booleano (encendido/apagado). La base con ranuras sugiere tres huecos en la memoria del programa.

    Declaración y asignación en JavaScript

    // Forma moderna: let (valor puede cambiar) o const (referencia fija)
let edad = 20;                    // número
const nombre = "María";           // cadena (const: no reasignas el nombre a otra cosa)
let esEstudiante = true;          // booleano

// Cambiar el valor de una variable ya declarada:
edad = 21;

// Mostrar en consola del navegador (F12 → Consola)
console.log(nombre, edad, esEstudiante);

    Pseudocódigo (declaración explícita de tipo)

    entero edad
cadena nombre
logico activo

edad <- 20
nombre <- "María"
activo <- verdadero

escribir nombre, edad, activo

    Aquí el tipo se declara primero; en JavaScript el tipo se “infiere” del valor que asignas.

    Cargando modelo 3D…

    Cómo leerlo: izquierda = número, centro = texto, derecha = sí/no. En RA, compara las formas: no son tres cajas iguales a propósito.

    Nivel 1 · 1.3 — Entrada, proceso y salida (el “esqueleto” de casi todo programa)

    En lógica de programación, conviene ver cualquier tarea como tres bloques conectados: entrada (qué datos llegan o lees), proceso (cálculos, decisiones, repeticiones) y salida (qué resultado muestras, guardas o devuelves). No es magia: es la misma idea que un diagrama de flujo que empieza, hace trabajo en el medio y termina en un resultado.

    En clase, la traza (seguir valores a mano, vuelta por vuelta) conecta este tema con el 3.3: antes de confiar en la computadora, tú simulas ser la máquina con papel y lápiz. En el navegador, console.log es una salida rápida para depurar y aprender.

    Ejemplo mínimo: rectángulo

    JavaScript

    // Entrada: medidas (podrían venir de un formulario más adelante)
let base = 4;
let altura = 5;

// Proceso: regla geométrica
let area = base * altura;

// Salida: ver el resultado
console.log("Área:", area);

    Modelo 3D: reutilizamos el flujo del tema 1.1 para recordar que “arriba” suele ser el inicio y los datos bajan hacia acciones y resultado.

    Cargando modelo 3D…

    Mismo escenario que 1.1: piensa “datos arriba → decisión o cálculo → caminos o resultado abajo”. En RA, úsalo para explicar en voz alta entrada / proceso / salida.

    Nivel 2 · 2.1 — Operadores y expresiones

    Una expresión es una “frase matemática o lógica” en código: combina valores (números fijos, variables, textos) con operadores (símbolos que dicen qué hacer) y al evaluarse produce un resultado. El modelo 3D sugiere: valor A → operación → valor B → resultado.

    En este curso los ejemplos usan JavaScript (el lenguaje del navegador). Si casi no has programado, aquí estás aprendiendo sintaxis nueva: no hay print como en otros lenguajes; lo habitual para “mostrar” algo al probar es console.log(...) en la consola del navegador (tecla F12 → pestaña Consola en Chrome o Edge).

    Lo mínimo de JavaScript antes de los operadores

    Piensa en una instrucción como una orden que el intérprete ejecuta de arriba abajo. Para escribir código que tenga sentido necesitas estas piezas:

    • Declarar una variable (reservar un nombre y, a la vez, darle un valor inicial): en JavaScript moderno suele usarse let nombre = valor; cuando el valor puede cambiar después, o const nombre = valor; cuando no vas a reasignar ese nombre a otra cosa. El símbolo = aquí no es “igual en matemáticas”: es asignación (“guarda esto en esta variable”).
    • Punto y coma (;): en JavaScript muchas veces es opcional al final de una línea, pero es buena costumbre ponerlo para separar instrucciones con claridad (sobre todo si vienes de otro lenguaje donde es obligatorio). Si lo omites, el motor intenta “cerrar” la instrucción solo; hasta que cojas práctica, puedes terminar cada instrucción con ;.
    • “Imprimir” o ver un resultado: en el navegador no escribes print. Usas console.log(expresión); — lo que pongas entre paréntesis se calcula y aparece en la consola. Puedes mostrar texto entre comillas: console.log("Hola"); o el valor de variables y expresiones: console.log(a + b);
    • Comentarios (el ordenador los ignora): línea con // o bloque /* ... */. En los ejemplos de abajo, los // explican qué hace cada línea.

    Tu primer mini-programa en JavaScript (consola)

    // 1) Declaro una variable y le asigno un número
let puntos = 10;

// 2) Otra variable
let bonus = 5;

// 3) Una expresión: suma (aquí aún no usamos todos los operadores del tema)
let total = puntos + bonus;

// 4) Muestro el resultado en la consola del navegador (F12 → Consola)

    Tipos comunes de operadores

    Operadores aritméticos en JavaScript

    Sirven para calcular con números. En código no se usa la × ni la ÷ del teclado: se escriben * y /. El orden de las operaciones sigue las reglas de matemáticas; usa paréntesis ( ) si quieres forzar un orden concreto.

    Símbolo Nombre Ejemplo Resultado / idea clave
    + Suma 7 + 2 9. Con dos números, los suma.
    + Concatenación "Hola" + " Ana" "Hola Ana". Si intervienen textos (strings), + los une. Mezclar número y texto puede dar resultados raros: "5" + 3 → "53".
    - Resta 7 - 2 5. Resta el segundo valor al primero.
    - Signo (unario) -(-5)
    let x = 4; -x    		 5 y -4. Un - delante de una expresión invierte el signo.
    * Multiplicación 4 * 5 20. Equivale a “por” en matemáticas.
    / División 10 / 4 2.5. En JavaScript la división puede dar decimales (no es solo “parte entera”).
    % Módulo (resto) 10 % 3
    7 % 2    		 1 y 1. Es lo que sobra al dividir el primer número entre el segundo (división entera). Muy usado para saber si un número es par: n % 2 === 0.
    ** Potencia 2 ** 3 8 (2³). Base a la izquierda, exponente a la derecha. También verás Math.pow(2, 3) en ejemplos avanzados.

    Resumen rápido: + − * / son como en álgebra (con * y / en el teclado). % no es “por ciento” en código: es el resto de una división. ** es potencia. Si mezclas tipos (texto y número), JavaScript puede convertirlos de formas sorprendentes: mientras aprendes, trabaja con números puros o usa console.log para comprobar.

    • Comparación (dan un booleano): === igual estricto (mismo valor y tipo), !== distinto, < > <= >=. En JavaScript moderno se prefiere === frente a == para evitar conversiones raras.
    • Lógicos: && (Y: ambas partes verdaderas), || (O: al menos una verdadera), ! (NO: invierte true/false).
    • Asignación: = guarda un valor en una variable. No confundir con ===, que compara.

    Precedencia y paréntesis: como en matemáticas, lo que está entre ( ) se evalúa primero. Si dudas, añade paréntesis: hace el código más claro para ti y para quien lo lea.

    Nivel 2 · 2.2 — Decisiones múltiples (else if, anidación y tablas de decisión)

    El if / else del tema 1.1 sirve para dos caminos. En la vida real suele haber varias categorías: nota a letra, menú con opciones, rangos de descuento, etc. En JavaScript encadenas condiciones con else if: “si no fue la primera, prueba la segunda; si no, la tercera…”.

Ojo con la legibilidad: muchos niveles seguidos confunden; a veces una tabla en papel o un switch (cuando comparas un valor contra constantes) ayuda — en este curso basta con entender la lógica; el sintaxis exacto de switch la verás con práctica.

  console.log("Letra C");

    Nivel 2 · 2.4 — Estructuras repetitivas (bucles)

    A veces no quieres escribir la misma instrucción 100 veces. Un bucle (o loop) le dice a la computadora: “vuelve a hacer este trozo de código” según una regla. Cada vuelta se llama iteración. En el modelo 3D, el centro apilado es el cuerpo del bucle (lo que se repite); el anillo es volver al inicio; las flechas son “otra vuelta más”, en orden.

    Los ejemplos están en JavaScript. Para probarlos, abre la consola del navegador (F12 → pestaña Consola), pega un bloque y mira lo que imprime console.log. Cada instrucción puede terminar en ; para que sea más fácil de leer.

    Por qué los bucles suelen costar al principio

    Con un if el programa mira una vez y elige un camino. Con un bucle, la misma zona del código se ejecuta varias veces y hay que imaginar “el tiempo pasando”: el valor de un contador cambia, la suma crece, etc. Eso exige un hábito nuevo: seguir el código con el dedo o en una libreta, vuelta por vuelta, hasta que encaje la idea.

    • ¿Qué se repite? Lo que va entre llaves { } del bucle (el cuerpo).
    • ¿Cuándo paro? Cuando la condición deja de cumplirse (en while) o cuando el contador llega al tope (en for).
    • ¿Qué cambia en cada vuelta? Casi siempre un número (n, i, k) o un acumulador (suma). Si nada cambia hacia la salida, el bucle puede no terminar nunca.

    Analogía rápida

    Piensa en subir escaleras: en cada peldaño repites “subir un paso” hasta que la condición “aún no llegué al piso” sea falsa. O en dar vueltas a la manzana contando: “vuelta 1, vuelta 2, vuelta 3… hasta 10”. El bucle es la regla; el contador es el número de vuelta.

    while — “mientras se cumpla esto, sigue”

    Antes de cada vuelta se mira la condición entre paréntesis. Si es verdadera, se entra en las llaves, se hace el trabajo, y al llegar al cierre } se vuelve arriba a preguntar otra vez por el while. Si la condición es falsa, no se entra (o se sale) y el programa sigue después del bloque.

    Regla de oro con while: dentro del cuerpo debe pasar algo que acerca el estado a que la condición falle (por ejemplo subir n de uno en uno). Si no, la condición sigue siempre verdadera y el navegador se queda dando vueltas (bucle infinito) — no es un mensaje rojo siempre; a veces solo “se congela” o tarda mucho.

    Estructura de un while (léelo de arriba abajo)

    // Objetivo: mostrar en consola los números 1, 2 y 3

let n = 1;                    // ① Empezamos en 1 (estado inicial)

while (n <= 3) {              // ② ¿Seguimos? Solo si n es menor o igual que 3
  console.log("Vuelta:", n);  // ③ Trabajo de esta vuelta: mostrar n
  n = n + 1;                  // ④ Acercamos n al “no cumplir” (1→2→3→4)
}                             // ⑤ Al salir del while, n ya vale 4; no se imprime 4

// Cuando n pasa a 4, la condición n <= 3 es falsa y el programa sigue debajo del while

    La línea n = n + 1; es la que hace que el bucle no dure para siempre. También puedes escribir n++; (significa lo mismo en este contexto).

    for — las tres piezas en la primera línea

    El for agrupa en un solo sitio lo que en un while repartes: cómo empiezas, cuándo paras y qué haces al terminar cada vuelta.

    for ( A ; B ; C ) { … }

    • A — se hace una sola vez al inicio (crear el contador, ej. let i = 1).
    • B — pregunta antes de cada vuelta: si es falsa, se sale del for (ej. i <= 3).
    • C — se hace al final de cada vuelta, antes de volver a mirar B (ej. i++ para pasar al siguiente número).

    Mismo recorrido 1…3 con for

    // let i = 1  → empiezo en 1
// i <= 3     → sigo mientras i no pase de 3
// i++        → al final de cada vuelta, subo i en 1

for (let i = 1; i <= 3; i++) {
  console.log("i vale", i);
}

// Salida en consola: 1, luego 2, luego 3

    Patrón muy usado: acumular una suma

    Muchos ejercicios piden “sumar del 1 al 10” o “del 1 al 100”. La idea es siempre la misma: una variable suma que empieza en 0 y, en cada vuelta, le sumas el número de turno (suma = suma + k o abreviado suma += k).

    Sumar del 1 al 100 con for

    let suma = 0;                       // todavía no hemos sumado nada

for (let k = 1; k <= 100; k++) {
  suma = suma + k;                  // en esta vuelta sumamos k (1, luego 2, … 100)
}

console.log("Suma del 1 al 100:", suma);   // debe salir 5050

    Si tu reto pide solo hasta 10, cambia 100 por 10 y el resultado esperado será 55.

    Errores habituales (y cómo detectarlos)

    • Olvidar subir el contador en un while → la condición nunca cambia → bucle infinito.
    • Condición mal escrita (ej. i < 3 cuando querías incluir el 3) → faltan o sobran vueltas. Relee el enunciado: ¿“hasta 10” incluye el 10? Entonces suele hacer falta <= 10.
    • Mezclar el primer valor del contador: si empiezas en 0 o en 1 cambia cuántas vueltas hay; no está mal usar 0, pero entonces la condición debe coincidir con lo que pide el ejercicio.
    • No inicializar la suma en 0 antes del bucle → el primer suma + k parte de un valor basura.

    Resumen en un solo bloque (while + for + suma)

    // --- while: del 1 al 3 ---
let n = 1;
while (n <= 3) {
  console.log("Vuelta número:", n);
  n = n + 1;
}

// --- for: mismo recorrido ---
for (let i = 1; i <= 3; i++) {
  console.log("i vale", i);
}

// --- suma 1 + … + 100 ---
let suma = 0;
for (let k = 1; k <= 100; k++) {
  suma = suma + k;
}
console.log("Suma total:", suma);

    entero n
n <- 1
mientras n <= 3 hacer
  escribir n
  n <- n + 1
fin mientras

para i desde 1 hasta 3 hacer
  escribir i
fin para

    Nivel 3 · 3.1 — Arreglos y listas (colecciones ordenadas)

    Una variable normal guarda un valor. Un arreglo (array) o lista ordenada guarda varios valores en fila, cada uno en una “casilla” numerada. A ese número de casilla se le llama índice. En JavaScript, C, Java, Python y muchos otros, el primer índice es 0, no 1: la primera casilla es [0], la segunda [1], etc.

    La longitud del arreglo (length en JavaScript) dice cuántos elementos hay. El último índice válido es length - 1. Si pides un índice que no existe, obtienes undefined (u error en otros lenguajes).

    ¿Para qué sirve en lógica?

    Procesar muchas notas, nombres de una clase, días de la semana, etc., sin crear 50 variables sueltas. Suele combinarse con un for que recorre cada índice de 0 a length - 1. En el modelo 3D, las casillas suben como escalera: la más baja es el índice 0 y cada una siguiente es el siguiente índice; los postes delanteros crecen en altura para recordar el orden 0, 1, 2, 3.

    JavaScript: crear, leer, modificar, recorrer

    // Declaración con corchetes (literal de arreglo)
let notas = [85, 90, 72, 88];

// Leer por índice
console.log(notas[0]);   // 85  (primera posición)
console.log(notas[3]);   // 88  (cuarta posición)

// Cantidad de elementos
console.log(notas.length);  // 4

// Cambiar un elemento
notas[2] = 95;           // subí la tercera nota

// Recorrer todo el arreglo con for
for (let i = 0; i < notas.length; i++) {
  console.log("Índice", i, "→ valor", notas[i]);
}

    arreglo notas de enteros
notas <- [85, 90, 72, 88]

para i desde 0 hasta longitud(notas) - 1 hacer
  escribir "Casilla", i, ":", notas[i]
fin para

    Nivel 3 · 3.2 — Funciones y modularidad

    Escribir todo el programa en una sola lista larga es difícil de leer y de corregir. Una función es un bloque con nombre que agrupa pasos para una tarea concreta (por ejemplo “calcular el precio con IVA” o “validar la edad”). Puedes llamar la función varias veces desde distintos sitios sin copiar y pegar el código.

    Partes de una función

    • Nombre — describe la acción (calcularTotal, saludar).
    • Parámetros — valores de entrada que la función necesita (van entre paréntesis). Son como variables locales que solo existen dentro de la función.
    • Cuerpo — instrucciones entre llaves { }.
    • Retorno (return) — opcional: devuelve un valor al lugar donde llamaste la función. Si no hay return, la función puede servir solo para efectos secundarios (ej. mostrar un mensaje).

    El modelo 3D es una mini fábrica: embudo azul = datos que entran (argumentos); tubos grises = pasillo hacia el proceso; caja oscura con “tapa” = interior de la función (donde ocurre el cálculo, sin que quien llama vea cada paso); embudo verde = resultado que sale (return).

    JavaScript: declarar y usar funciones

    // Función con un parámetro y valor de retorno
function cuadrado(x) {
  return x * x;
}

let resultado = cuadrado(5);   // resultado es 25

// Varios parámetros
function sumar(a, b) {
  return a + b;
}

console.log(sumar(10, 3));     // 13

// Función que usa operadores y condicional dentro
function puedeVotar(edad) {
  if (edad >= 18) {
    return true;
  }
  return false;
}

// Forma compacta equivalente (flecha) — verás esto en muchos tutoriales
const duplicar = (n) => n * 2;
console.log(duplicar(4));      // 8

    Las variables a, b, x solo “viven” dentro de cada función: son parámetros o variables locales.

    Pseudocódigo

    función sumar(a, b): entero
  retornar a + b
fin función

función puedeVotar(edad): logico
  si edad >= 18 entonces
    retornar verdadero
  si no
    retornar falso
  fin si
fin función

entero total
total <- sumar(10, 3)
escribir total

    Nivel 3 · 3.3 — Pruebas mentales, trazado y comentarios
